# 디자인 기록 (DESIGN.md)

이 문서는 말씀캠프 PWA의 디자인 의사결정 과정과 디자인 시스템을 정리한 기록이다. UI를 어떤 기준으로 만들었고, 무엇을 시도했으며, 왜 최종안을 선택했는지를 담는다.

---

## 1. 디자인 목표와 원칙

- **모던 미니멀, 화이트 베이스**: 정보(공지·일정·말씀)가 우선이므로 장식을 절제하고 여백과 가독성을 중심에 둔다.
- **브랜드 일관성**: 참가자는 캠프 포스터를 먼저 접한 뒤 앱을 설치한다. 앱이 포스터와 같은 색·무드를 공유해야 하나의 캠페인으로 인식된다.
- **모바일 우선**: 80~100명이 폰으로 사용하므로 모든 결정의 기준은 좁은 화면과 작은 글씨의 가독성이다.

---

## 2. 색상 결정 과정

### 1차: 바질 그린
초기에는 차분한 바질 그린을 포인트 색으로 화이트 베이스에 적용했다. 캠프·자연의 분위기에는 맞았으나, 캠프의 실제 브랜드를 반영한 색은 아니었다.

### 2차: 포스터 톤으로 전환
캠프 포스터를 확인한 결과, 실제 정체성은 **맑은 하늘빛·아쿠아 파스텔 + 흰 빛 + 금빛 반짝임**의 서정적인 수채화 톤이었다. 그린은 포스터와 이질적이어서, 브랜드 일관성을 위해 전체 색을 포스터 계열로 전환했다.

### 최종 팔레트
파스텔을 그대로 UI에 쓰면 대비가 부족해, 포인트는 한 단계 진한 톤을 사용했다.

| 용도 | 값 |
|---|---|
| 강조 / 활성 / 링크 | `#3F7D99` |
| 제목 · 공지 제목 | `#2F5E72` |
| 카드 배경 | `#F2F8FA` |
| 테두리 | `#D4E6EC` |
| 본문 | `#21343C` |
| 보조 텍스트 | `#52707D` |
| 흐린 텍스트 | `#8AA6B3` |

## 3. 타이포그래피 결정 과정

영문 표어 "Where are you?"를 세리프 이탤릭으로 두기로 하면서, 한글 폰트가 이와 어울리는지가 관건이었다. 다음 순서로 탐색했다.

1. **Pretendard / Spoqa Han Sans Neo** — 깔끔한 산세리프. 그러나 세리프 영문 표어와 결이 맞지 않았다.
2. **Nanum Myeongjo** — 한글·영문을 명조로 통일했으나 전통적인 붓터치가 강해 올드하게 보였다.
3. **Hahmlet** — 모던 고대비 세리프. 우아하나 다소 각졌다.
4. **Gowun Batang (최종)** — 부드럽고 둥근 바탕체. 포스터의 포근한 무드와 가장 잘 맞고, 영문 세리프와도 자연스럽게 어울린다.

### 결정 사항
- 전체 폰트를 **Gowun Batang**으로 통일한다.
- 제목은 가는 무게(weight 400)로 두어 포스터의 가늘고 우아한 인상을 따른다.
- 제목과 공지 제목의 색은 `#2F5E72`로 통일한다.
- 영문 표어는 세리프 이탤릭으로 포스터의 포인트를 살린다.

## 4. 레이아웃 결정

- **수채화 헤더**: 홈과 각 페이지 상단에 CSS 그라데이션(라디얼 + 리니어)으로 수채화 질감을 표현하고, 금빛 반짝임을 절제해서 배치한다. 명도와 채도는 "진하되 밝게"를 기준으로 조정했다.
- **주제 말씀 정렬**: 가운데 정렬과 왼쪽 정렬을 비교한 결과, 2줄 본문은 왼쪽 정렬이 읽기 편하고 앱 전체의 좌측 정렬 리듬과 일관되어 **왼쪽 정렬**을 택했다. 가운데 정렬은 한 줄짜리 전시형 문구에 적합하다.
- **공지 UX**: 홈에는 최신 2개만 노출하고 우측 상단에 상대시간(just now / N분 전)을 표시한다. 공지 클릭 시 상세 페이지로, 섹션의 "전체 보기"로 전체 목록 페이지로 이동한다.
- **하단 탭 순서**: 홈 / 일정 / 말씀 / 방배정 / 사진. 사진은 라우트가 아니라 외부 Google Photos 공유 앨범으로 연결한다.

## 5. 앱 아이콘

초기 아이콘은 그린 테마를 따랐으나, 전체 색을 포스터의 블루 테마로 전환하면서 아이콘도 함께 재디자인했다.

- 흰 배경 위에 밝은 청색 `#67BDDB` 반짝임 심볼을 두어 포스터의 청색·금빛 반짝임 모티프를 단순화했다.
- 그린 계열 흔적을 제거해 앱 전체 색과 홈 화면 아이콘의 인상을 일치시켰다.
- 단순한 심볼 하나로 작은 크기에서도 또렷하게 보이게 했다.

---

## 6. 스플래시 화면

앱을 열 때 빈 화면이 잠깐 보이는 대신, 브랜드를 각인시키는 진입 연출을 넣었다.

- 콜드 스타트마다 흰 배경 전체화면 위에서 아이콘 → 로딩 막대 → 홈 순서로 이어진다.
- 1단계는 아이콘만 보여 브랜드를 먼저 인지시키고, 2단계에서 아이콘이 작아지며 위로 올라가고 그 아래 로딩 막대가 브랜드 그라데이션으로 채워진다.
- 로딩 막대는 실제 진행률이 아니라 진입을 매끄럽게 잇는 연출 장치이며, 다 채워진 뒤 페이드아웃으로 홈에 자연스럽게 넘어간다.
- 타이밍은 상수로 분리해 조정할 수 있게 했다. 아이콘 노출과 막대 채움 시간을 넉넉히 두어 급하지 않고 차분한 인상을 주도록 맞췄다.

## 7. 디자인과 기술의 교차점에서 마주친 문제

### iOS PWA 상태바
- iOS는 `theme-color` 메타가 있으면 그 값으로 상태바를 단색으로 칠한다. 헤더의 수채화 그라데이션을 상태바까지 잇기 위해서는 이 동작을 고려해야 한다.
- `apple-mobile-web-app-status-bar-style`은 **홈 화면에 추가하는 시점에 한 번만 읽힌다.** 따라서 스타일을 바꾼 뒤에는 앱을 삭제하고 재설치해야 적용된다.
- Android는 상태바가 OS가 칠하는 단색 영역이라 그라데이션이 불가능하며, 매니페스트 `theme_color`를 그라데이션 상단 색과 맞춰 자연스럽게 잇는 것이 최선이다.

### 다이어그램 렌더링
- README의 아키텍처 다이어그램을 Mermaid로 작성했으나 GitHub에서 렌더링이 깨졌다. 라벨의 특수문자·줄바꿈에 민감해 안정적이지 않았다.
- 어디서든 동일하게 보이도록 다이어그램을 **PNG 이미지로 교체**해 안정성을 확보했다.

## 8. 디자인 시스템 요약

- **색**: 위 팔레트 표 참조. Tailwind 팔레트 한 곳에서 관리해 페이지 전체에 일괄 적용한다.
- **폰트**: Gowun Batang(한글·영문), 영문 표어는 세리프 이탤릭.
- **카드**: 흰 배경, 1px 테두리(`#D4E6EC`), 둥근 모서리. 강조 카드는 2px 테두리.
- **헤더**: 수채화 그라데이션 배경 + eyebrow + 제목.
- **여백**: 모바일 가독성을 위해 충분한 상하 여백을 둔다.

---

## 9. 향후 개선

- 작은 UI 텍스트(공지 본문·하단 탭)의 가독성을 폰에서 점검하고, 필요 시 해당 부분만 산세리프로 분리한다.
- 포스터 원본 수채화 이미지를 헤더 배경으로 사용해 질감을 더 살린다.
- 다크 모드 대응을 검토한다.
