# 공지사항 첨부파일 및 이미지 지원 설계 연구

## 1. 목표
- 공지사항 본문 중간에 이미지 삽입이 가능하도록 한다.
- PDF 파일을 공지에 첨부하고, 상세 화면에서 열람할 수 있게 한다.
- 이후 Rich Text Editor 도입 시에도 데이터 구조를 크게 바꾸지 않도록 설계한다.

## 2. 아키텍처
### 저장소
- 이미지와 PDF 같은 파일은 Firestore에 직접 저장하지 않는다.
- Firebase Storage에 업로드한 뒤, 다운로드 URL을 Firestore에 저장한다.

### 데이터 모델
- 공지 문서 필드:
  - title: 문자열
  - body: 문자열
  - attachments: [{ name, url }]
  - createdAt: Timestamp
- 본문 이미지 삽입은 Markdown 형식으로 처리한다.
  - 예: `![캠프 안내](https://...)`

### 프론트엔드 처리 흐름
1. 관리자가 이미지/파일을 선택한다.
2. 파일을 Storage로 업로드한다.
3. 업로드 완료 후 URL을 얻는다.
4. 본문 문자열 또는 attachments 배열에 반영한다.

## 3. 현재 구현 방향
- 관리자 화면에서 이미지 추가와 PDF 첨부 버튼을 제공한다.
- 이미지는 본문 텍스트에 Markdown 이미지 문법으로 삽입한다.
- PDF는 공지 하단의 첨부파일 목록에 표시한다.
- 상세 화면에서 이미지와 첨부파일 링크를 렌더링한다.

## 4. 확장성 고려
- 향후 React Quill, TipTap 같은 Rich Text Editor로 교체해도, Storage 업로드 로직은 재사용 가능하다.
- 본문은 문자열 기반이므로, 나중에 HTML 기반 포맷으로 전환해도 최소한의 수정으로 대응 가능하다.
- 첨부파일은 별도 배열로 분리되어 있어 관리가 쉽다.
